# Phase A: Information Architecture

## Target Structure

### Core Handbook: 15 Sections (Primary Reading Flow)

The handbook divides into 4 conceptual groups, each with distinct purpose:

---

## GROUP 1: FOUNDATION (Sections 01-03)

Establishes brand philosophy, market position, and behavioral pillars.

### 01 Philosophy
**Purpose:** Define what Vayasya believes and why it exists.
**Content type:** Manifesto + principle statements
**Presentation:** Full-width manifesto band + principle cards
**Key elements:**
- Mission statement
- Core beliefs (3-5)
- What we stand for / What we reject
- Do/Don't examples

### 02 Positioning
**Purpose:** Define how Vayasya communicates its value.
**Content type:** Approved positioning statements + copy blocks
**Presentation:** Statement plates with copy-ready text
**Key elements:**
- Elevator pitch (15 words)
- Full positioning statement (50 words)
- Vertical-specific positioning
- Proof points
- Do/Don't examples

### 03 Operating Pillars
**Purpose:** Define the 4 behavioral pillars that guide all work.
**Content type:** Pillar definitions + sayable/unsayable guidance
**Presentation:** Pillar lanes (vertical strips) with "Say this / Avoid this"
**Key elements:**
- 4 pillars with descriptions
- Sayable phrases per pillar
- Phrases to avoid per pillar
- Do/Don't examples

---

## GROUP 2: VISUAL SYSTEM (Sections 04-07)

Governs all visual identity elements.

### 04 Logo Usage
**Purpose:** Protect logo integrity.
**Content type:** Logo variants + usage rules + violations
**Presentation:** Correct vs. incorrect visual plates
**Key elements:**
- Master logo lockup
- Vertical logos
- Clear space requirements
- Size minimums
- Correct placement examples
- Forbidden treatments (visual gallery)
- Do/Don't examples

### 05 Color Palette
**Purpose:** Define and constrain color usage.
**Content type:** Color tokens + usage ratios + scenarios
**Presentation:** Swatch grid with token names + usage examples
**Key elements:**
- Core tokens (background, foreground, muted, border)
- Brand gold usage
- Vertical accent colors
- Semantic colors (success, warning, info, danger)
- Usage ratios (primary/secondary/accent)
- Context-specific scenarios
- Do/Don't examples

### 06 Typography
**Purpose:** Define typographic system.
**Content type:** Font stacks + hierarchy + specimens
**Presentation:** Live type specimen ladder
**Key elements:**
- Primary font (Hind) specifications
- Monospace font (JetBrains Mono) usage
- Type hierarchy (H1-H6, body, caption)
- Weight usage guidelines
- Line height and spacing
- Wordmark typography rules
- Do/Don't examples

### 07 Imagery
**Purpose:** Guide visual asset selection.
**Content type:** Preferred vs. forbidden image styles
**Presentation:** Image comparison board (preferred/forbidden)
**Key elements:**
- Photography style guidelines
- Illustration approach
- Icon usage
- Image don'ts (stock clichés, etc.)
- Do/Don't examples

---

## GROUP 3: COMMUNICATION (Sections 08-10)

Governs language, tone, and claims.

### 08 Voice & Tone
**Purpose:** Define brand personality and language rules.
**Content type:** Persona definitions + tone spectrum + terminology
**Presentation:** Persona cards + rewrite examples + terminology table
**Key elements:**
- Brand voice attributes (3-4)
- Tone spectrum (formal ↔ casual, etc.)
- Context-specific tone adjustments
- Terminology dictionary (approved terms)
- Banned phrases list
- Sentence rewrite examples
- Do/Don't examples

### 09 Claims Discipline
**Purpose:** Prevent legal/ethical risk in marketing claims.
**Content type:** Claim classification + evidence requirements
**Presentation:** Evidence matrix + proof checklist
**Key elements:**
- Claim types (aspirational, directional, measured, contractual)
- Evidence requirements per type
- Legal-safe patterns
- Risk indicators
- Approval requirements by claim type
- Do/Don't examples

### 10 Writing Mechanics
**Purpose:** Standardize style and formatting.
**Content type:** Mechanical rules + examples
**Presentation:** Mechanics rail with live examples
**Key elements:**
- Capitalization rules
- Punctuation standards
- Number formatting
- Date/time formatting
- List formatting
- Abbreviation rules
- Common errors to avoid
- Do/Don't examples

---

## Reading Flow

The handbook should read top-to-bottom in one continuous scroll:

```
[Hero: Brand identity summary]
    ↓
[Section 01-03: Foundation]
    ↓
[Section 04-07: Visual System]
    ↓
[Section 08-10: Communication]
    ↓
[Section 11-15: Application]
    ↓
[Appendix: Reference material]
    ↓
[Footer: Version + ownership]
```

Each section should be self-contained and immediately readable without requiring tab/accordion interaction.

---

## Content Hierarchy Per Section

Every section follows this internal hierarchy:

```
1. Section number + title (H2)
2. One-line summary
3. Intent statement (why this matters)
4. Core rules (immediately visible)
5. Primary content (tables/grids/examples - no tabs)
6. Do/Don't examples (visible)
7. [Optional] Expanded details (single accordion if needed)
8. Template references (if applicable)
```

Maximum hierarchy depth: 2 levels before actionable guidance appears.
